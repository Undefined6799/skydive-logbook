## Gotchas for the next agent

1. **The dev sandbox ships Python 3.10; the project targets 3.11+
   (D15).** If `python -m pytest` fails with
   `cannot import name 'UTC' from 'datetime'` or
   `cannot import name 'StrEnum' from 'enum'`, the sandbox is too old.
   Create a venv: `uv venv .venv311 --python 3.11 && uv pip install
   --python .venv311/bin/python -e ".[dev]"`. Then
   `.venv311/bin/python -m pytest backend/tests`.

2. **Canonical venv name is `.venv`.** `.venv-sandbox` and
   `.venv-test` may still exist on disk — past sessions' agents
   couldn't delete them from sandboxed shells. Run
   `rm -rf .venv-sandbox .venv-test` once locally to clean up.
   They're in `.gitignore` (the `.venv-*` pattern from 2026-04-29
   Slice 1) so they don't leak into commits.

3. **`atomic_write_stream` is the storage primitive for any
   streaming write path** (attachment-edit when it ships). Returns
   `(sha256, size)` from a single pass over the bytes — no second
   read for hashing.

4. **D26 drop-and-reindex lives at `open_index`, not in a migration
   framework.** Bumping `INDEX_SCHEMA_VERSION` is the only mechanism.
   `main.py` runs `reindex_from_xml` synchronously after a rebuild
   per ARCH-1; failure refuses startup. Schema bumps mid-stream:
   v6 added rig_id (R.2.2-light.d.1, D33), v7 added is_tandem +
   jumper_credentials (D47, Phase D.4). Index is at v8 today.

5. **The subprocess crash harness (`_crash_child.py`) is reusable.**
   The audit's TEST-1 / TEST-2 slices extend it with crash points for
   `update_jump`, `delete_jump`, and dropzone CRUD. Don't rewrite it
   from scratch.

6. **D17 canonical timestamp form is ISO-8601 UTC with ms precision
   + `'Z'`.** Every new code path emitting timestamps must use
   `_now_utc_iso()` (or equivalent), never
   `datetime.now().isoformat()`. `backend/storage/trash.py` is the
   one known offender today (audit CODE-3); fix when convenient.

7. **User isolation at the service layer returns 404, not 403, on
   wrong-user access.** Existence is never leaked. Preserve this on
   every new service function.

8. **RFC 9457 problem+json is the only error envelope (D16).** Any
   service-layer error must be a `ServiceError` subclass so it
   bubbles through `on_service_error`. The catch-all
   `@app.exception_handler(Exception)` wraps non-ServiceError raises
   into `InternalServerError` so the wire shape stays
   `application/problem+json`.

9. **D48: no auth surface in v0.1.** `bearerAuth` is gone from
   OpenAPI; `Settings.api_key` does not exist; `get_user_id()`
   always returns `"default"` (D8). Re-adding any of these requires
   a successor D-entry pinning the LAN/multi-user enforcement story.

10. **D49: cloud-sync (Dropbox/iCloud/OneDrive) is supported best-
    effort.** WAL-on-cloud is documented-incompatible upstream
    (https://www.sqlite.org/wal.html §8); the D26 reindex flow is
    the recovery path. Don't switch to `journal_mode = DELETE`.

11. **D50: intra-process writes are serialised by a writer
    lock at the service layer.** ARCH-4 shipped on 2026-04-29.
    Service writes go through `with_writer_lock`; reads of rename-
    prone resources also acquire the lock so they don't race a
    rename mid-list.

12. **Pyright is the third gate**, alongside `pytest` and `ruff`. CI
    enforces it on every push and PR. D51 pins the policy: strict
    for production code, basic for `tests/`, lxml at the typed
    boundary in `backend/xml/`. Suppression form is
    `# pyright: ignore[<rule>]  # <reason>` only — never
    `# type: ignore` (the mypy form).

13a. **Vitest is the frontend gate.** `npm test` from `frontend/`
    runs the smoke suite under jsdom (`vitest run`). CI has a
    dedicated `frontend` job (Node 20, ubuntu-latest). The smoke
    suite is import-drift detection only — not a behaviour suite.
    Tests live in `frontend/test/views.smoke.test.jsx`; config at
    `frontend/vitest.config.js` (separate from `vite.config.js`).

13b. **PyInstaller spec at project root.** `skydive-logbook.spec`
    pulls version from `pyproject.toml`, datas the XSDs + README
    template + `frontend/dist/`. **Not bake-tested** — needs a real
    per-platform build before v0.1 ship. See `BUILD.md` for the
    commands and the verification gaps.

13c. **D52 is DRAFT.** Code-signing posture for v0.1: ad-hoc /
    unsigned by default, signed before GA. Per-developer certs are
    NOT in the repo; PyInstaller spec exposes the hooks via env
    vars. Finalise D52 after the first beta release.

13. **Console scripts:** `skydive-logbook` (REST only),
    `skydive-logbook-desktop` (full launcher with frontend build +
    pywebview window). Both defined in `pyproject.toml [project.scripts]`.

14. **Two ports.** `backend.main` defaults to `127.0.0.1:8765` per
    `Settings.bind_port`. `skydive-logbook-desktop` overrides to
    `127.0.0.1:8000` per `_BACKEND_PORT` in `launch_desktop.py`.
    The two-port discrepancy is real but isn't part of this audit's
    scope; document any user-visible behavior carefully.
