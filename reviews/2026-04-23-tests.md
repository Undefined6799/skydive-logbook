## Invariant test coverage checklist

- **D2 XSD validation refusal:** `PARTIAL` — hardened parser tests are comprehensive (DTD/entity rejection, size limits), but **no test verifies that an XSD-invalid but well-formed XML document is rejected** on write. The validator module exists and is imported by test_validator.py; a test calling `validate(jump)` with a semantically broken Jump should be added.

- **D2 hardened parser rejects DTD/entity/external:** `PASSING` — test_validator.py lines 33–77 extensively test this (forbid_dtd=True, billion-laughs, XXE file://, plain DOCTYPE).

- **D3 reindex from XML only:** `UNTESTED` — reindex script is unimplemented (backend/scripts/reindex.py line 9). Once implemented, this must have a dedicated test proving the index can be rebuilt from XML on disk after a fresh drop.

- **D4 path safety / NFC normalization:** `PARTIAL` — test_filesystem.py tests sanitize functions in isolation; test_create_jump.py implicitly exercises them. No explicit round-trip test verifying NFC normalization survives a logbook reopen.

- **D10 atomic_write fsync + crash-between:** `MOSTLY PASSING` — test_filesystem.py tests crash during stream (lines 294–326), old-file preservation (lines 310–326), temp cleanup. Does not explicitly verify fsync() was called (low priority; actual filesystem semantics are reliable).

- **D16 RFC 9457 shape assertion:** `PASSING` — test_errors.py lines 144–201 extensively test problem+json content-type, member structure (type, title, status, detail, instance, code, request_id, errors), OpenAPI schema registration. Excellent coverage.

- **Multi-file write crash coverage:** `PASSING` — test_crash_recovery.py §D25 rows A–D and test_manifest.py combined cover: (1) atomic_write jump.xml before manifest, (2) atomic_write_stream for attachments before jump.xml, (3) folder_reconcile recovery. All four crash points per D25 have explicit subprocess-kill tests.

## Summary

**Suite is mostly sound.** Filesystem, XML parsing, error responses, crash recovery, and manifest logic are well-tested with real temp directories and service-layer calls. The biggest gaps are:

1. **D3 reindex from XML is untested because the feature is not yet implemented** (per task #4, backend/scripts/reindex.py line 9). This is a blocking test debt — once reindex lands, a test proving "drop index, rebuild from XML, verify correctness" must follow.

2. **XSD validation on write is not explicitly tested** — hardened parser tests are excellent, but test_validator.py does not verify that an XSD-invalid Jump model is rejected by validator.validate().

3. **Stale-lock recovery and D9 interrupt semantics are not tested** — test_lockfile.py is minimal (happy path + contention timeout only).

4. **Minor:** Unused monkeypatch parameter in test_create_jump.py:245 suggests incomplete refactoring.

**D3's "rebuild from XML" guarantee cannot be verified in its current form because reindex is deferred.** Once task #4 is resolved, add a dedicated test to backend/tests/test_reindex.py proving the round-trip: XML → drop index → rebuild → correct rows.
